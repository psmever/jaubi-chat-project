# 0순위 - 적용 승인 룰

이 규칙은 다른 모든 User Rule, Project Rule, "바로 구현해줘" 요청보다 우선한다.

사용자 메시지에 `적용해`라는 단어가 없으면 AI assistant는 무조건 제안만 한다.

`고쳐줘`, `수정해줘`, `구현해줘`, `만들어줘`, `추가해줘`, `커밋해줘`, `푸시해` 같은 표현만으로는 직접 실행하지 않는다.

AI assistant의 기본 역할은 다음으로 제한한다.

- 코드 구조 제안
- 코드 예시 제시
- 설계 초안 작성
- 문제 원인 분석
- 에러 로그 해석
- 테스트 방법 안내
- 리뷰와 개선점 제안
- 사용자가 작성한 코드 검토

파일 생성, 수정, 삭제, 포맷팅, 자동 수정, 빌드 산출물 생성, 테스트 실행, git add, commit, push 등 워킹트리나 이력에 영향을 줄 수 있는 모든 작업은 `적용해`가 있을 때만 수행한다.

읽기 전용 작업은 허용한다.

- 파일 조회
- 코드 검색
- `git status`
- `git diff`
- `git log`
- 설정, 문서, 테스트 코드 분석

`적용해`가 포함된 경우에만 직전 제안 또는 현재 문맥상 명확한 범위 안에서 직접 작업한다.

범위가 불명확하면 실행 전에 한 번 확인한다.

커밋은 `적용해 커밋해줘`처럼 파일 반영 승인과 커밋 승인이 함께 명확할 때만 수행한다.

커밋 제목과 본문은 한글을 기본으로 하고, 형식은 `type(scope): 한글 제목`을 사용한다.

푸시는 사용자가 `푸시해`라고 말했더라도 즉시 실행하지 않는다.

푸시 직전에는 대상 브랜치, 원격 저장소, 커밋 범위를 요약하고 한 번 더 확인받은 뒤에만 `git push`를 실행한다.
